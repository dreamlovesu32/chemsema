## 总结

今天的工作把编辑器从“以键编辑为主”继续推进到更完整的化学绘图表面。最大的一条主线是新增箭头工具，包括直箭头、弯箭头、端点编辑、hover 控制点、接近 ChemDraw 的箭头样式、叉号/双斜杠 no-go 装饰，以及真正平滑的圆弧渲染。

另一条主线是编辑器基础设施：文档单位明确为点数（`pt`），历史记录增加语义命令层，剪贴板操作进入 engine 和 viewer，前端通过重新生成的 wasm 绑定暴露更多编辑能力。

## 文档单位

文件格式现在明确记录文档单位：

- 格式文档新增 `format.unit = "pt"`；
- engine 的单位工具补齐点数相关换算；
- viewer 的单位工具从 CSS 像素映射到点数，不再把世界单位当成抽象厘米；
- 多处命中半径、聚焦半径、默认键参数和粘贴偏移改为基于点数的尺度。

这样存储几何更接近 ChemDraw/CDXML 坐标，也消除了之前“抽象文档单位”的歧义。

## 命令历史

engine 下新增了命令层：

- `EditorCommand` 记录语义操作，例如添加键、添加箭头、应用箭头样式、剪切、粘贴、插入模板、移动选区、旋转选区、应用文本编辑、替换端点标签等；
- `HistoryEntry` 保存命令以及精确的 before/after 文档快照；
- 拖动类命令会在手势过程中刷新最终 `after` 状态，让 redo 回到鼠标松开时的位置；
- 新增 `docs/editor-command-history.md`，说明命令表面，以及“已提交文档变化”和“临时交互状态”的区别。

这让撤销/重做有了更清晰的语义历史路径，同时当前仍然保留精确文档恢复能力。

## 剪贴板

engine 增加内部剪贴板支持：

- 复制选中的节点、标签和选中的键；
- 剪切作为一个可撤销命令；
- 粘贴时生成新的节点和键 id；
- 粘贴内容按固定点数偏移；
- 粘贴后刷新相连标签和选择边界；
- 通过 wasm 和 viewer 工具栏/快捷键路径暴露复制、剪切、粘贴。

这里的剪贴板是编辑器内部数据，不是浏览器文本剪贴板。

## 箭头工具

箭头工具是今天最大的功能块。

### 工具入口和绘制

左侧工具栏在文本工具下方新增箭头工具。绘制行为如下：

- 任意点按下；
- 拖到任意点；
- 松手创建箭头；
- 不按 `Alt` 时方向按 15 度吸附；
- 按住 `Alt` 时自由方向；
- 鼠标松开点就是箭头真实末端，箭头头部长度包含在箭头长度内部。

新画出的箭头不会自动选中，用户可以连续继续绘制。

### 工具栏控制

箭头工具栏现在包括：

- 普通实心箭头；
- 弯箭头；
- 镜像弯箭头；
- 空心箭头；
- 未封闭空心箭头；
- 箭头大小：大、中、小，默认小；
- 端点样式组：直线、head 全箭头、tail 全箭头、head-left 半箭头、head-right 半箭头、tail-left 半箭头、tail-right 半箭头；
- no-go 样式组：带叉箭头、带双斜杠箭头；
- 加粗箭头。

工具栏按箭头类型、弯曲弧度、箭头大小、端点样式、no-go 装饰和加粗分组，中间用竖线分隔。

### 箭头样式和 payload

箭头对象现在保存结构化 `arrowHead` payload：

- `kind`：`solid`、`curved`、`curved-mirror`、`hollow` 或 `open`；
- `curve`：带符号的弯曲角度；
- `head` 和 `tail`：`none`、`full`、`half-left` 或 `half-right`；
- `length`、`centerLength`、`width`；
- `bold`；
- `noGo`：`none`、`cross` 或 `hash`。

CDXML 转换器也做了修正：只有源 CDXML 的 head/tail 端点设置真实存在且不是 false-like 值时，导入箭头才启用对应端点。

### 直箭头和弯箭头渲染

渲染现在支持：

- 实心填充箭头头；
- 半箭头保持尖端锋利，并避免主线穿过箭头尖端；
- 空心和未封闭空心箭头轮廓；
- 弯曲实心箭头用 SVG path 渲染；
- 叉号和双斜杠 no-go 标记，并根据当前箭头大小缩放；
- 加粗箭头变体。

新增 `RenderPrimitive::Path`，让弯箭头主线可以作为 path 渲染，而不是用可见 polyline 近似。

### 弯箭头修复

今天修掉了一个严重的弯箭头渲染错误。圆弧采样点本身是对的，但三次贝塞尔近似圆弧的控制点系数写错了：

- 错误：`4 / 3 * tan(delta / 2)`；
- 正确：`4 / 3 * tan(delta / 4)`。

错误系数会把控制点推得过远，导致 270 度弯箭头像圆角矩形或几段直线拼出来。修正后 path 会贴近真正圆弧。新增了回归测试，检查首个控制点不会远离圆弧切线。

### Hover 和编辑

选择工具和箭头工具都支持箭头 hover：

- 未选中的箭头会显示端点、中心和箭头头部侧点；
- 没有箭头头时只显示起点、中心、终点；
- 半箭头只显示对应一侧的控制点；
- 双头箭头两端都会显示侧点；
- 已选中的箭头不显示 hover，使用正常选中框。

在选择模式下：

- 点击 hover 箭头可以选中；
- 拖动 hover 的 head/tail 端点可以修改端点；
- 拖动 hover 中心点可以修改弯曲角度；
- 端点拖动不按 `Alt` 时按 15 度吸附；
- 弧度拖动不按 `Alt` 时也按 15 度吸附；
- 弧度和旋转角度标签现在显示整数度数。

箭头工具下也可以使用 hover 端点/弧度拖动能力，但不会因此选中箭头。

## 选择和对象交互

选择行为扩展到箭头和更丰富的对象操作：

- 箭头对象可以被选择工具选中；
- 箭头选中框会把 path primitive 纳入边界计算；
- 选区移动、临时对象拖动、端点移动和旋转继续接入命令历史；
- 已选中对象会压制可能和选中框操作冲突的 hover 控件；
- 旋转角度标签改为整数。

选择 overlay 新增箭头中心 halo 和圆形 hover 控制点样式。

- 回归脚本适配点数单位尺度；
- 文本编辑控制器调整为和 engine 状态保持一致。
