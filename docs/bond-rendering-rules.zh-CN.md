# Chemcore 键绘制规则

本文档记录当前 Rust 渲染内核中的键绘制和键接触规则。目标不是描述实现细节，而是固定“哪些几何规则是定义，哪些只是实现手段”。后续重构 `render` 模块时，应以本文档作为行为基线。

本文档中的 “应 / 必须 / 不得” 都是规范性要求；“可以 / 允许” 是实现可选空间。

## 目标

- 键绘制应由 Rust 内核统一定义，不依赖 viewer 的二次修补。
- 同类键应尽量共享几何模型，而不是按键型堆局部启发式。
- 节点接触应优先由轮廓和端点 profile 决定，而不是由 SVG `linecap` 或前台样式偶然决定。
- 预览态和落地态应复用同一套几何。

## 适用范围

- 本文档只约束 fragment 路径。
- `legacy molblock` 路径只要求兼容输出，不要求逐条满足本文档中的新几何规则。
- 本文档描述的是 2D 几何输出，不讨论前台交互状态管理、hover 装饰或选中框样式。

## 术语

- 主键：决定键接触和节点占位的主轴母体。
- 副线：单侧双键和三键中，相对主键偏移的附属线。
- 母体：未被 knockout 切段之前的完整 polygon。
- 端点 profile：某根键在某个端点真正吃掉的轮廓链，最终参与 polygon 收口。
- 保持原样：保持该键标准母体外框，不为了接触去改端帽轮廓。
- 退让：不改外框拓扑，只沿轴向缩短或后退，使该键不压进接触对象。
- 同侧：相对于节点局部法向，副线或外线位于与当前参考线相同的一侧。
- 直通：两根轴线近似 `180°` 反向共线的情况。

## 总体分类

当前键分成三类：

1. 有主键的对象。
2. 无主键但有固定母体形状的对象。
3. legacy molblock 路径。

本文档只讨论 fragment 路径中的前两类。

### 有主键的对象

- 普通单键
- 粗实键
- 普通虚键
- 哈希键
- 单侧双键
- 中心双键
- 三键
- 实锲形键
- 虚锲形键

### 统一几何口径

- 所有实键都按 polygon 处理，而不是 `line`。
- 普通单键和粗实键几何本质相同，只是宽度不同。
- 实锲形键按梯形处理，不按三角形处理。窄端是一条很短的边，而不是退化点。
- 普通虚键是“母体 polygon + 白色 knockout 切段”。
- 哈希键是“粗实母体矩形 + 更密的白色 knockout 切段”。
- 虚锲形键是“实锲形母体梯形 + 白色 knockout 切段”。

## ChemDraw 绘图模板参数

Default 和 ACS Document 1996 不是同一套几何参数的简单缩放结果，而是两套模板。切换模板时，已有键和后续新键都应同步切换模板参数。

当前对齐 ChemDraw 的关键差异：

| 参数 | Default | ACS Document 1996 | 说明 |
| --- | ---: | ---: | --- |
| 键长 | `30.0pt` | `14.4pt` | 切换模板时按该比例缩放现有结构几何 |
| 普通线宽 | `1.0pt` | `0.6pt` | 用于普通键、窄端锲形键和图形线宽 |
| 粗键宽 | `4.0pt` | `2.0pt` | 粗实键模板宽度 |
| 实/虚锲形宽端 | `6.0pt` | `3.0pt` | 固定模板值，不随键长按比例推导 |
| label clip 退让 | `1.35pt` | `0.95pt` | 键端从 label glyph/box 退开的额外距离 |
| hash spacing | `2.9pt` | `2.5pt` | hash 家族的模板间距 |
| bond spacing | `12%` | `18%` | 双键内间距公式使用实际键长和该百分比 |

锲形键宽端和 label clip 退让都属于模板参数，不应从键长反推。默认模板和 ACS 模板可以同时保持各自固定宽度、固定退让，并在切换时由 engine 重写 bond-level 字段。

## 统一判定顺序

每个 bond 端点都应按以下顺序求最终 geometry：

1. 先确定该键的标准母体形状。
2. 如果节点有 label，先执行 label clipping。
3. 再决定该端点是否进入节点接触规则。
4. 如果进入接触，再按键型选择：
   - 主键求交
   - 副线求交
   - 只退让
   - 保持原样
5. 在最终母体上叠加 dash/hash/knockout。
6. 预览态与落地态必须走同一套流程。

任何实现都不得把 viewer 的 stroke、line join、line cap 当成几何定义的一部分。

## 主键接触

“主键接触”指接触区由主键轮廓决定，而不是由副线、虚线段或 viewer 描边决定。

### 两根键接触

适用于普通单键、粗实键、普通虚键、哈希键、单侧双键主键、三键主键、实锲形键主轴、虚锲形键主轴等“有主键”的情况。

规则：

1. 两根键在节点处共享一个中心点。
2. 每根键在该端点都有两条主键轮廓线。
3. 计算两根键的 `inner-inner` 交点和 `outer-outer` 交点。
4. 这两个交点组成接触区边界。
5. 每根键吃自己的端点 profile，直接生成自己的 polygon，不靠额外遮罩补丁修形。

补充：

- 这里的交点允许使用延长线，不要求交点落在线段原始长度内。
- 如果角度极小，允许使用 miter limit 约束。
- 如果因为 miter limit 被截断，则每根键必须仍然拥有落在自身轮廓延长线上的裁切点，不得退回到任意经验中点。

### 三根及以上键接触

规则：

1. 先按节点周向顺序排列 incident bond。
2. 只处理相邻 pair，不处理所有组合。
3. 每一对相邻键的内侧轮廓线求交，形成节点周围的一圈交点。
4. 每根键吃自己那一段端点轮廓链。
5. 三键节点中，实键可能自然变成五边形；更多键时，顶点数继续增加。

补充：

- 多键节点只处理相邻 pair，不处理所有组合。
- 非相邻 pair 的交点不得用于节点中心拓扑。
- 如果某类键被定义为“保持原样”，则该键不参与节点中心占位；由其他键退让。

### 直通例外

- `180°` 直通不强求交点。
- centered double 在夹角大于 `162°` 时不延长。

## 单键与粗实键

- 普通单键和粗实键都按四边形渲染。
- 二者只在宽度参数上不同。
- 接触时，它们都属于主键接触对象。

## 普通虚键

- 普通虚键的母体仍然是一根标准矩形主键。
- 白色切段沿主轴等距布置。
- 接触以母体矩形为准，而不是以切开的黑段为准。
- 黑色段长度可以随 dash 规则变化；普通虚键不要求像 hash 家族那样“黑段严格等长”。

## 哈希键

哈希键不是“普通虚键加粗”，而是单独一类：

- 母体始终是粗实矩形。
- 黑色段长度必须一致。
- 白色段允许在一定范围内波动。
- 只有当总长超出当前段数允许的范围时，才增减白色段数量。
- 黑色段的长度优先级高于白色段的等距性。

### 哈希键与普通主键

- 保持标准矩形，不主动变形去贴其他键。
- 如果节点是多键节点，其他键退让，哈希键保持原样。

### 哈希键与 centered double

- 不求交，不改矩形端帽。
- 只沿哈希键自身中心线退让。
- 退让参考是 centered double 正确那一侧的外线。
- 白色切段跟随退让后的实际矩形，不沿旧长度切。

## 实锲形键

- 母体是梯形。
- 宽端宽度固定，窄端宽度固定为很短的顶边。
- 宽端在普通主键接触中允许按轮廓求交变形。
- 宽端如遇 label，应优先避免压入 label。
- 长度变化时，应优先保持端宽定义，再更新梯形侧边，而不是重新按比例缩放整个母体。

## 虚锲形键

- 母体是标准梯形，不是由若干独立线段直接定义。
- 黑色段沿梯形主轴排布。
- 黑色段长度保持一致。
- 白色段允许在范围内波动，超范围才调整数量。

### 虚锲形键与普通主键

- 保持标准梯形，不主动求交改宽端。
- 多键节点中，优先保持自身母体形状，由其他键退让。

### 虚锲形键与 centered double

- 不宽端求交。
- 只沿虚锲形自身轴线退让。
- 宽端宽度保持固定，因此长度变化后梯形会自然变形。
- 白色切段必须跟随退让后的实际梯形，而不是按退让前长度切。

## 单侧双键

单侧双键由一根主键和一根副线组成。

### 主键

- 主键属于普通主键接触对象。
- 与单键、粗实键、三键、楔形等按主键接触规则处理。

### 副线

- 外副线默认保持原样，不参与普通接触。
- 内副线才参与接触求交。
- terminal 端与主键等长。
- 非 terminal 端按“与主键间距 * `sqrt(3) / 3`”缩短。
- 当主键与连接主键夹角小于 `90°` 时，副线允许退让；但如果对方存在同侧副线，则同侧副线仍然优先求交。

### 锐角退让

- 如果主键连接的是主键，且该侧夹角小于 `90°`，副线可退让。
- 但如果对方也有同侧副线，则同侧副线仍然直接求交。
- 如果对方副线不在同侧，则按退让处理。
- 如果遇到 centered double，则以 centered double 的中心线作为退让参考。

## centered double

centered double 不是一个整体 patch，而是两根独立子线。

规则：

- 每根子线各自判断各自端点。
- 每端、每侧独立决定是否延长到其他键的轮廓。
- 大于 `162°` 的近直通情况不延长。
- `180°` 直通保持原样。
- 与 hash 家族接触时，centered double 是被参考的一方，不反向改 hash 家族外框。
- 适用于：
  - 实实中心双键
  - 虚实中心双键
  - 双虚中心双键

## 三键

- 三键主线属于主键接触对象。
- 两根外线各自独立判断自己的端点 profile。
- terminal 端外线保持全长。
- 与单侧双键副线类似，外线接触按各自轮廓判断。

## 多键节点中的 hash 家族优先级

若一个节点同时接多根键，并且其中有 `hash bond` 或 `hashed wedge`：

- `hash bond` / `hashed wedge` 保持原样。
- 其他键退让。
- 故意留一点白，不做完全无缝贴合。

## 判定优先级

同一个端点如果同时命中多条规则，应按以下优先级处理：

1. label clipping
2. hash 家族保持原样规则
3. centered double 近直通不延长规则
4. 主键接触求交
5. 单侧双键副线同侧求交
6. 单侧双键副线锐角退让
7. terminal 等长 / 端部固定长度规则
8. dash / hash knockout 切段

后面的规则不得推翻前面已经确定的几何拓扑，只能在剩余自由度内继续收紧。

## Label 与裁切

- 节点有可见 label 时，优先先做 label clipping。
- label clipping 发生在 bond contact 之前。
- 键最终 geometry 应同时满足 label 裁切和节点接触规则。

## 预览

- 拖动画键时不画假的 overlay bond。
- 预览态通过临时文档调用同一套 `render_document`。
- 因此预览和落地必须共享同一套接触几何。

## 重构约束

后续拆分 `render` 模块时，应保持以下约束：

- 不改变本文档中的几何规则。
- 优先拆高层调度与低层几何，不在一次重构里继续改规则。
- 每次拆分后必须通过：
  - `cargo test`
  - `npm run build:engine-wasm`

## 测试基线

以下几类规则必须至少有一层自动化测试覆盖：

- 主键接触侧别判定和直通判定。
- centered double 的 `162°` 阈值。
- miter limit 截断后的求交回退。
- hash 家族“黑段等长、白段浮动”的节奏分配。
- boundary pairing 不得错误选择 trivial assignment。
- 多键节点中 hash 家族保持原样、其他键退让。

建议测试分层：

- 低层单测：直接测求交、侧别、节奏分配 helper。
- 渲染级测试：测输出 primitive 的数量、类型、关键顶点关系。
- 前台验证：只负责确认 viewer 没有 stroke / AA 伪影，不负责定义几何。

## 非目标

- 不在本文档中描述 SVG 抗锯齿细节。
- 不保证不同浏览器像素级完全一致。
- 不把 debug 面板、调试 SVG、开发期 patch 当成正式渲染特性。

## 当前已知适合拆分的职责边界

- `render_document` 和 object-level 调度
- bond render 高层分发
- main bond contact kernel
- bond geometry / intersection helpers
- render primitive 定义与 push helper
- legacy mol 渲染兼容层
