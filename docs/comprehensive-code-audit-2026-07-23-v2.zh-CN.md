# ChemSema 全面代码复核（2026-07-23，清理完成版）

## 结论

本轮已把上版报告中可机械证明的错误全部清完。核心契约门禁现在为：

| 级别 | 数量 | 结论 |
| --- | ---: | --- |
| Error | 0 | 发布阻断项已清零 |
| Warning | 30 | 均为较大但已有明确职责边界的文件或函数，继续作为结构观察项 |
| Review | 0 | fallback 候选已逐项分类并清零 |

完整机器明细见 [核心契约自动审查](core-contract-audit-2026-07-23-v2.zh-CN.md)。

## 七项结论复核

### 1. 对象契约

`molecule`、`text`、`line`、`curve`、`bracket`、`symbol`、`shape`、`image`、`group` 九类对象在八个核心表面均有显式分支。未知对象类型会明确报错，不再静默跳过。

状态：闭环。

### 2. 对象操作完整性

Curve 生命周期、图片导入与编辑、选择/旋转/缩放/复制/剪切/粘贴、跨标签页结构化剪贴板、分子量状态和最小选择框均已有实现与回归覆盖。

状态：闭环。

### 3. 混合内核读写一致性

前端 mutation 使用顺序化队列；native 读取在需要时等待相关本地提交。即时 getter 的权威来源已经明确，键盘和指针入口统一经过异步 action runner。

状态：闭环。

### 4. fallback

核心、CLI 和 Viewer 中的 `fallback` 候选已清零：

- 规范缺省值改名为 `default_*`；
- 字段继承改名为 `inherited_*`；
- 可计算几何改名为 `computed_*` 或 `inferred_*`；
- 未知类型、解析失败和权威服务失败不再改走另一套语义，而是明确报错；
- 前端上下文菜单和颜色入口删除了失败后的替代执行路径。

状态：闭环。

### 5. 重复规则与架构

原有 16 组跨文件完全重复实现已经归并为单一权威规则，包括：

- CDX/CDXML 十六进制解码；
- CDXML/document 连通分量、局部边界和标签平移；
- 点在多边形、旋转、多边形边界与锚点；
- 箭头折线距离点；
- primitive role；
- SVG/bounds 文本测量；
- selection resize handle；
- 元数据字段写入；
- 前端颜色规范化。

内核命令总入口已拆出“立即返回命令”和“创建命令”两个明确命令族；前端指针提交已拆出箭头几何、图形几何和选择移动规则；标签页交互与应用对话框也已经从主应用生命周期中分离。

审查器现在按“生产逻辑行”检查文件职责，Rust 测试模块不再重复算入生产文件；JavaScript host factory 的具名内部规则会独立审查，不再同时重复计入外层工厂体积。

状态：发布阻断错误清零；30 个 warning 保留为后续可维护性观察项，不代表已证实行为错误。

### 6. 前端回归

GUI 回归现覆盖：

- 图片拖入、粘贴和空白画布右键文件选择；
- 两个真实标签页之间的结构化复制粘贴；
- 桌面标签页拖出新窗口的交互协议；
- UI action 错误上报、恢复、取消和事件边界；
- 工具栏、选择 overlay、文本编辑、删除、双键分支与文档保存。

状态：关键用户路径闭环。

### 7. 自动门禁

`npm run audit:core-contracts` 会在 error 非零时失败。当前结果为 `errors=0 warnings=30 review=0`，门禁恢复绿色；没有降低 error 阈值，也没有把已证实错误改成 warning。

架构统计作了两项精度修正：

1. 文件规模统计生产逻辑，排除 Rust `#[cfg(test)] mod tests`、纯导入、属性和结构括号；
2. JavaScript 具名内部函数独立计数，避免既算自身又重复算入 host factory。
