# ChemSema 全面代码复审（2026-07-23，第二版）

## 总结

本轮按上一版报告的七项主要结论重新检查，并直接修复了能够确定规则的问题。

结论不是“全部完成”。对象契约、Curve 操作闭环、混合内核读写屏障、已确认的语义 fallback 和关键 GUI 链路已经修复；架构拆分、重复规则、前端异步入口以及剩余 fallback 命名候选仍未清零。因此核心契约门禁目前仍应失败，不能作为发布绿灯。

机器审查覆盖 141 个源文件、4769 个函数，结果为：

| 级别 | 数量 | 含义 |
| --- | ---: | --- |
| Error | 12 | 2 个超大文件、10 个超大函数，违反当前架构阈值 |
| Warning | 53 | 23 个较大函数、13 个较大文件、16 组完全重复函数、1 个桌面标签页拖出测试缺口 |
| Review | 55 | 名称或代码形态含 fallback，必须逐项用规范或版本证据分类 |

## 七项结论复审

### 1. 对象类型没有单一契约：核心问题已修复

已建立内核唯一对象类型枚举 `SceneObjectKind`，当前正式类型为：

`molecule`、`text`、`line`、`curve`、`bracket`、`symbol`、`shape`、`image`、`group`。

JSON 进入内核时递归校验对象类型，未知类型不再静默跳过；渲染、选择覆盖、可选择性、全选、剪贴板完整性、旋转、变换策略和 CDXML 导出均改为枚举穷尽分派。CCJS 中缺失的 `curve` 和 `symbol` 说明也已补齐。

机器能力矩阵目前为 9 类对象 × 8 个表面全部 explicit。这里的“explicit”表示存在明确分支，不代表所有对象的每个高级语义均已穷尽。

状态：核心契约闭环。

### 2. Curve 是半实现对象：已修复并建立生命周期测试

Curve 现在具有明确边界、命中测试、框选、全选、移动、旋转、缩放、复制、剪切、粘贴、删除、上下文样式和 CDXML 导出行为。

新增 Curve 完整生命周期回归，并增加未知对象类型 JSON 必须报错的回归。测试不再只验证“能导入和能画”，而是验证它能走完编辑器对象的完整操作链。

状态：本轮审查范围内闭环。

### 3. 混合 WASM/native 状态没有读写屏障：已修复关键链路

已建立顺序化操作队列和 native 读取屏障。需要读取 native 状态的剪贴板 API 会先等待影响该读取的本地 mutation 提交；颜色、样式、撤销/重做等即时 UI getter 明确读取本地权威状态。

键盘编辑命令现在会在首次异步文件快捷键判断之前同步入队，避免快速连续快捷键被重排。完整 GUI 回归暴露出的另一个独立错误也已修复：粘贴完整文档后会形成多个 molecule 对象，而删除逻辑过去只修改当前 fragment，没有删除 `selection.molecule_objects`。现在会删除所有选中的 molecule/graphic 对象，并清理失去引用的资源。

实际 GUI 链路“全选 → 复制 → 粘贴 → 全选 → 剪切”已通过；桌面混合内核 250 ms 人工延迟回归也通过。

状态：关键一致性规则闭环；后续新增 native getter 必须继续声明读取权威和屏障要求。

### 4. 已确认的语义 fallback：已删除或改为有证据的明确分支

本轮处理了四类已确认问题：

- 普通 CDXML 缺少全部 `n@p` 时不再凭拓扑任意布局，而是明确报错。
- 仅对 `CreationProgram="CDXMLWriter"` 的 topology-only 文件保留具名兼容规则。
- 拥有嵌套 fragment connection table 的父节点缺少 `p` 时，只依据其 `ExternalConnectionPoint` 语义定位；无足够连接方向证据时不猜。
- 标签键退让只使用字形轮廓或明确扩张后的字形轮廓；文本框是布局数据，不再作为墨迹轮廓替代品。
- Viewer 侧删除了未知字形轮廓推断和另一套文本宽度估算，字形几何回到内核单一权威。
- native 后台队列和图片解码中的吞异常分支改为显式错误上报。

需要特别说明：机器审查仍找到 55 个 `FALLBACK-NAMED` 候选。这些不是 55 个已证明的错误；其中可能包含规范默认值、格式兼容分支或测试术语，但每一项仍要链接字段规范、版本证据或行为测试后才能改名并从复核清单移除。

状态：已确认的硬 fallback 已处理；55 个候选仍待逐项证据化。

### 5. 重复规则和大函数导致行为漂移：尚未完成

当前仍有 16 组跨文件完全重复函数，包括：

- CDX/CDXML 十六进制颜色或值解码；
- CDXML 与 document 的连通分量、边界和标签平移；
- 编辑与渲染的箭头距离点计算；
- 多处 polygon bounds、anchor、point-in-polygon；
- render、engine、SVG 的 primitive role；
- render bounds 与 SVG 的文本宽度计算；
- selection resize geometry；
- 前端颜色规范化。

架构门禁的 12 个 error 全部来自超大文件或超大函数。最突出的是 `Engine::execute_command`（583 行）、`viewer/app.js`（4074 行）、`createEditorPointerController`（1582 行），以及文档流、视口、overlay、文本编辑器、上下文菜单、标签页和窗口生命周期的大闭包。

这些问题没有适合在一次规则修复中机械拆分。拆分必须以状态机和所有权边界为单位，同时保持历史、revision、selection 和错误语义不变。

状态：未完成，是当前最大结构性风险。

### 6. 前端回归覆盖不足、异步错误不统一：异步入口已收口，端到端仍有缺口

GUI 端到端回归现在覆盖：

- 打开和另存为 CCJS/CDXML/SVG；
- Ctrl+S 保存 CCJZ；
- 复制、粘贴、剪切；
- 图片拖入和图片粘贴；
- 插入图片后切到选择工具并产生选择框；
- 两个真实浏览器标签页之间的结构化剪贴板；
- 工具栏、光标、选择 overlay、删除工具、双键精确分支、缩放和样式。

本轮后续清理已经加入统一 UI action runner。它负责区分用户取消和真实失败、执行具名恢复回调、显示错误通知、写入诊断事件，并允许直接调用测试观察 rejection。此前机器扫描发现的 15 个未保护 async DOM 事件入口已经全部改接该 runner，当前计数为 0。

仍有两个明确缺口：

1. 桌面标签页拖出并生成新窗口没有端到端测试。
2. 图片“空白处右键打开文件选择器”的系统对话框边界没有在当前 GUI 回归中形成独立断言，不能用图片拖入/粘贴测试代替。

状态：异步事件错误入口闭环；系统窗口边界的端到端覆盖尚未闭环。

### 7. 自动门禁过去不能约束上述问题：已增强，但当前应保持红灯

审查脚本现在会：

- 从正式对象枚举核对对象集合；
- 检查 9 类对象在 8 个核心表面的显式分派；
- 检查未知类型静默分支和禁止命令；
- 扫描语义 fallback 候选；
- 检查混合 getter、读取屏障和操作测试；
- 检查前端异步事件、关键 GUI 测试缺口；
- 统计大文件、大函数和跨文件完全重复函数。

`npm run audit:core-contracts` 在存在 error 时返回非零。当前 12 个架构 error 尚未处理，因此门禁失败是正确结果，不应降低阈值或把它们改成 warning 来获得绿色状态。

状态：门禁能力已增强；仓库尚未达到门禁全绿。

## 本轮发现并修复的额外真实错误

完整工作区测试在删除 fallback 后暴露了两个单点测试不容易发现的规则洞：

1. 嵌套 Fragment 的父节点位置应来自内部 ExternalConnectionPoint，而不是普通缺坐标布局。
2. 原生标签可能只有 glyph polygons、没有单独的 expanded clip polygons；此时应直接使用 glyph polygons，不能退到文本框，也不能完全取消退让。

两条规则均已写成明确分支并由回归测试覆盖。

## 后续建议顺序

1. 补桌面标签页拖出新窗口和右键图片文件选择器的端到端测试。
2. 合并 16 组完全重复规则，优先处理几何、primitive role 和文本宽度，避免多渲染后端继续漂移。

4. 按 gesture/document lifecycle 状态机拆分前端大闭包。
5. 逐项处理 55 个 fallback 候选：规范默认值改名为 `default_*`/`inherited_*`；版本兼容分支写明生产者或版本；无证据的替代算法删除并报错。
