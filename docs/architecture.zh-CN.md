# chemcore 架构

## 目标

`chemcore` 的定位是一个长期存在的化学文档核心，供以下场景共享：

- 浏览器宿主
- 桌面宿主
- 导入 / 导出链路
- 未来的编辑工具

这个项目不应该为了一个临时前端去优化。它应该从一开始就围绕最终核心架构设计，即使实现速度会慢一些。

## 核心原则

### 1. 平台无关的核心优先

最重要的资产是文档模型，而不是第一个 renderer。

核心层必须先定义清楚：

- 文档结构
- 对象身份
- 坐标系统
- 样式引用
- 分组和 z-order
- 带化学语义的对象
- 渲染约定

Web 和桌面只是宿主，不是两个独立产品。

### 2. 化学语义和文档语义分离

化学结构数据和文档对象数据解决的是两类不同问题。

化学语义包括：

- 原子
- 键
- 立体化学
- 分子缩写
- `molblock2d`

文档语义包括：

- 对象定位
- 分组
- 样式引用
- 文本框
- 箭头
- 可见性
- z-order
- 变换

架构上不应该把这两类关注点硬塞进同一个对象模型。

### 3. 文件格式稳定，运行时模型为执行优化

文件格式是持久化约定。

运行时场景模型是执行模型。

二者应该接近，但不需要完全相同。文件格式应该显式、可版本化、便于迁移。运行时模型则应该适合：

- 命中测试
- 局部重绘
- 选择
- 命令执行
- 撤销 / 重做

### 4. 渲染后端可替换

第一个 backend 可以是 web，但绘图 API 不应该假设 DOM、React 或任何浏览器专属原语。

长期可能需要的 backend 包括：

- SVG
- Canvas / WebGL
- 原生桌面渲染
- PDF / SVG 导出渲染器

### 5. 导入是一级子系统

`chemcore` 必须能吃进旧格式，尤其是 CDXML。

这意味着导入应该直接落到 `chemcore` 文档模型上，而不是先落到某种中间 UI 状态。

## 分层结构

目标系统按层拆分如下。

### Layer A: 文件格式

持久化后的 `chemcore` 文档。

职责：

- 版本管理
- 对象序列化
- 样式表序列化
- 对象关系
- 元数据

非目标：

- 运行时缓存
- 只属于 UI 的临时状态

### Layer B: 运行时文档模型

内存中的文档图。

职责：

- 按 id 查对象
- 父子关系
- 对象类型
- 变换
- 样式解析

这一层应该是确定性的，并且适合后端无关的渲染。

### Layer C: 场景与几何服务

Web 和桌面宿主都会用到的共享逻辑。

职责：

- 世界坐标
- 局部坐标
- 包围盒
- z-order 遍历
- 命中测试
- 变换组合
- 可见性判断

### Layer D: 渲染接口

后端无关的绘制约定。

接口至少需要支持：

- begin/end frame
- push/pop transform
- draw text
- draw line/path
- draw molecule
- apply style

接口不应该假设后端如何存储或绘制图元。

### Layer E: 宿主适配层

平台相关实现。

示例：

- web viewer
- desktop shell
- CLI exporter

宿主层不应该重新定义文档模型。

## 为什么这里要保留 CDXML 解析

CDXML 目前是最现实的导入入口，因为它能把基于 ChemDraw 的工作流接到未来的 `chemcore` 文档上。

迁移到 `src/chemcore/cdxml` 下的解析器之所以“暂时”，只是因为它属于导入子系统，而不是因为它是一次性代码。

它当前的职责是：

- 提取分子、文本、箭头、表格
- 通过 `molblock2d` 保留结构数据
- 提供足够信息，构造第一个 `chemcore` 文档转换器

## 第一阶段里程碑

第一阶段真正有意义的里程碑不是完整编辑。

而是：

1. `chemcore` file format v0.1
2. `chemcore` runtime model v0.1
3. 一个把提取出的 CDXML 数据转换进去的 converter
4. 一个能证明模型足够的只读 renderer backend

这个里程碑要回答的核心问题是：

“这个文档模型，能不能忠实表达我们要支持的那类化学页面？”

## v0.1 的非目标

下面这些内容明确不在第一版格式的范围内：

- 完整对齐 ChemDraw 功能
- 富 query chemistry
- 高级聚合物语义
- 完整反应语义
- 多页布局
- 协同编辑
- 二进制缓存格式

第一版应该优先优化清晰性、稳定性和可检查性。
