# CDX/CDXML 公共视觉回归规则（2026-07-20）

本轮修复遵守“字段定义优先，不按文件名、对象 ID 或单张图片坐标分支”的原则。

1. CDX `NodeType` 覆盖官方 0–13 全枚举；`BracketType` 按官方枚举值解码，不能把二进制整数直接当作 CDXML 文本。
2. `SupersededBy` 规范标签为 `0x0012`。读取兼容公共 ChemDraw 文件使用的生产者别名 `0x0013`，写出统一使用规范标签。该兼容只解决同义标签，不改变对象可见性规则。
3. 旧 CDX styled text 先按 UTF-8 解码；无效 UTF-8 才回退 Windows-1252。这样既不破坏现代文件，也能读取旧专利文件中的 `°`、`±` 等字符。
4. CDXML 键 `Order` 可以包含多个官方键级。没有显式可见 `query` 对象标签时，按键级集合合成 ChemDraw 查询助记文本，例如 `1 2` 显示为 `S/D`；位置由键中点、键方向和字号共同决定。
5. bracket graphic 的显式 `LineWidth` 优先；缺省时继承文档图形线宽，不能硬编码成固定 1 pt。
6. `BondSpacingAbs` 是多重键线段中心距的绝对值；它与 `BondSpacing` 同时存在时按官方规定优先，并按该键的实际端点距离换算成内核百分比表示。
7. CDX `BondSpacing` 的二进制单位是十分之一，不是整数点值；例如 `12.5` 编码为有符号整数 `125`，读取时必须除以 10。
8. `LineType`、`RectangleType` 和 `OvalType` 是位标志集合。虚线、粗线、填充、阴影等状态可以组合，不能按互斥枚举解码。
9. `ShadowSize` 不是绝对点数。CDXML 数值先除以 `100` 得到无量纲倍率，再乘对象的有效 `LineWidth`，才是阴影的几何偏移量。ChemDraw 静默 SVG 实测覆盖 `ShadowSize=100/200/300/400/600/800` 与 `LineWidth=0.6/1/2 pt`，18 个组合均满足 `offset = ShadowSize / 100 × LineWidth`；例如 `400 × 0.6 pt = 2.4 pt`。渲染时不能把 `400` 直接解释成固定 `4 pt`。
10. ChemDraw 的文字下划线不是浏览器随字体变化的默认装饰线。对公开 SVG oracle 中 101 条下划线复核后，7、10、12、14.45、16、18 pt 等字号均使用固定 `0.4 pt` 线宽，线中心位于字母基线下方 `0.8 pt`；线长等于该下划线 run 的实际排版宽度。内核必须把它输出为独立线段；浏览器的 SVG/CSS 下划线存在最小设备像素和字体相关行为，不能稳定表达这两个固定几何量，也不能沿用默认的约 `0.1em` 粗细和位置。
11. `face=96` 的 chemical script 以化学词元边界判断上下标，而不是只看整段文本末尾。数字跟在元素/基团后为下标；`+`/`-` 前为字母、数字或闭括号，且后方是字符串结尾或任意空白字符时，为该物种的上标电荷。因此 `Et3NH+ Cl-` 必须分成 `Et`、下标 `3`、`NH`、上标 `+`、普通 ` Cl`、上标 `-`。只把最后一个符号识别成电荷会系统性破坏由空格分隔的盐和多物种标签。
12. CDXML `GraphicType="Line"` 的线端采用方切端帽（`butt`）和尖角连接（`miter`），与箭杆一致；该规则不因 `LineType` 是否含 `Dashed` 或 `Bold` 而改变。圆帽是内核新建普通线的交互默认值，不能在导入边界覆盖源格式语义。
9. `ObjectTag` 的文本只有在对象语义本身要求显示，或明确写出 `Visible=yes` 时才绘制；未知标签不能因为带有文本就默认可见。
10. 旧版 `Graphic` 圆形若没有三维中心和轴端点，`BoundingBox` 按官方定义解释为两个定义点：第二点是圆心，第一点是圆周点；不能按普通矩形边界处理。
11. 多行节点标签必须保留逐行 styled-text runs。化学式中的 `+`/`-` 在换行前也属于行末电荷，导入层和标签几何刷新层必须使用同一分类规则。
12. `BeginAttach`/`EndAttach` 索引针对原始文本字符，而字形数组不含换行符。渲染时先把原始字符索引映射到可见字形索引；索引指向换行时锚定前一个可见字形，多行附件的纵坐标取目标字形所在行，不能复用首行基线。
13. 缺坐标的普通嵌套片段节点可以依据同片段内相连的已定位节点和文档键长推导位置；`ExternalConnectionPoint` 不参与这种推导，避免把连接占位点误画成实体节点。

每条规则都需要字段级测试和公共 ChemDraw 像素对照。门禁的局部缺陷跨度使用 ChemDraw 参考坐标中的固定单位，不除以画布面积或文件大小；大图不能稀释局部错误。只有双向覆盖率、固定跨度上限和相对连通组件匹配同时满足时，才允许把字体栅格化造成的细小差异归为等价。像素门禁中的旧“通过”若依赖过宽的拓扑兜底，而新结果的实际 IoU 和覆盖率更高，应修门禁分类规则，不能为保留旧 verdict 恢复错误绘制。

规范依据：

- IUPAC FAIRSpec CDX SDK：`Node_Type`、`Bracket_Type`、`Bond_Order`、`BondSpacingAbs`、`SupersededBy`。
- 真实 ChemDraw 21 保存的 CDX/CDXML 对照。
- 公共语料的 ChemDraw SVG oracle 与细节无关尺寸门禁。

## 2026-07-21 补充规则

14. 旧式 `ArrowType="FullHead"` 与现代 `ArrowheadHead="Full"` 分属两套枚举；现代 CDX 端点值为 `0=None`、`2=Full`、`3=HalfLeft`、`4=HalfRight`。导入时现代字段优先，不能把旧枚举名称套进现代字段，也不能把二进制 `2` 错读为半箭头。
15. 实心完整箭头的 `ArrowheadCenterSize` 同时控制尖端凹口深度和凹口附近的贝塞尔收束。ChemDraw 的内侧控制点半宽为凹口深度的约 `1/8`，并受外侧箭头半宽限制；不能按整个 `ArrowheadWidth` 的固定比例生成，否则浅凹口箭头会在尖端两翼明显鼓起。该公式与箭头方向无关，横向、纵向和斜向箭头共用。
16. ChemDraw SVG oracle 的 `matrix(a 0 0 a tx ty)` 明确给出 SVG 内部坐标变换；ChemDraw 的 CD 坐标以二十分之一文档单位写入该 SVG。视觉对齐器可将 `20a` 作为尺度候选，但最终仍须以实际墨迹重叠优于普通搜索结果为采用条件。门禁的近完全等价分支同时要求双向覆盖率不低于 `0.994`，且任一连通缺陷的跨度不超过 `15` 个参考单位、面积不超过 `18` 个参考平方单位；这些固定上限不得随画布或文件大小放宽。
17. `GraphicType="Line"` 的 `Head3D/Tail3D` 可以省略；此时 `BoundingBox` 的前两个数是头端点、后两个数是尾端点，而不是需要排序归一化的矩形范围。导入器必须保留这两对坐标并生成线对象；`SupersededBy` 指向替代对象的旧线仍然不渲染。
18. 没有 `BoundingBox` 和化学键的单节点 `fragment` 仍是分子对象，只要节点带显式 `Element`；不能把它降级成普通文本。无成键方向可供排版判断时，ChemDraw 会把第 16、17 族氢化物按常用分子式顺序显示为 `H2O`、`HCl` 等，即使 CDXML 化学文本仍按元素锚点顺序保存为 `OH2`、`ClH`。该反转只适用于零连接且文本确实匹配节点元素/显式氢数的第 16、17 族节点；第 15 族的 `NH3` 等保持元素在前，带键标签仍按连接方向决定。
19. 旧式 `GraphicType="Arc"` 的 `BoundingBox` 不是矩形范围：第一点是已写入的弧端点，第二点是圆心；另一个端点由该向量按 `AngularSize` 旋转得到。`AngularSize` 在 CDX 中以“度数乘 10”的 `INT16` 保存，在 CDXML 中使用度数原值。
20. `ArrowType` 是旧式基础类型与修饰位的组合；`NoGo`、`Dipole` 不得覆盖或吞掉 `FullHead` 等基础类型。现代 `NoGo` 的 CDX 枚举实测为 `2=Cross`、`3=Hash`，`Dipole` 为零长度布尔属性；偶极箭头渲染时在尾端沿箭杆方向偏移一个 `ArrowheadWidth`，再绘制总长为一个 `HeadSize` 的垂直横杠。

## 2026-07-23 补充规则

21. 任意 `<n>` 的直接子 `<fragment>` 都是该节点拥有的展开定义、连接定义或替代项缓存，不是页面上的独立可见分子。可见片段收集必须按这条结构关系判断，不能维护一份容易漏掉 `AnonymousAlternativeGroup`、`NamedAlternativeGroup`、`ElementListNickname` 等枚举值的白名单；节点自身的标签和外层连接仍正常显示。
22. `Order="1.5"` 表示芳香/离域的化学语义，不等于“自动补一条平行实线”。可见线道完全由 `Display`/`Display2` 决定：未写 `Display2` 时只有一个线道，该线道可由 `Display` 设为虚线；显式写出 `Display2` 时才生成第二线道，并采用它指定的样式。导出仍保留 `Order="1.5"`。
23. 多行节点标签一旦被 `BeginAttach`/`EndAttach` 按字符挂接，源文件的文本 `p` 和 `BoundingBox` 就是附件几何的权威坐标。导入后的标签刷新可以重建字形与退让多边形，但不得把标签重新流排到节点周围；绝对坐标必须减去所属场景对象的累计平移后写入资源局部坐标。右对齐和居中标签的 SVG 文本锚点仍使用 `p`，字形几何的行起点则取包围盒左边界。
24. `CurveType` 是位标志集合；线型、填充、闭合与箭头修饰可以组合。`CurvePoints` 的外层首尾点是 ChemDraw 的导向点，三次贝塞尔可见端点来自相邻控制点，不能把所有点当作逐点折线，也不能丢掉导向点。水平 curly bracket 的 `BoundingBox` 同样保留端点次序，并通过对象旋转表达方向。
25. ChemDraw 的 chemical-script 缩放在已测字体、字号和符号组合中稳定为 `0.75`；基线偏移需要按字体族处理，而不是只按 `face` 位或统一比例猜测。Times New Roman 的 7–8 pt 小字号以及 Calibri 使用各自的实测上下标偏移；Arial、Helvetica 等未触发专门规则的字体继续使用公共基线函数。该规则同时用于 SVG 文本、字形裁剪和原语几何，避免显示与退让采用不同基线。
