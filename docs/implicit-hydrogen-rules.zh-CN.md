# Chemcore 隐式氢规则

本文档定义 Rust engine 对节点标签自动加氢的行为基线。这里的“隐式氢”指写入节点 `numHydrogens`，并把简单元素标签显示为 `NH3`、`NH2`、`OH` 这类带氢形式。

## 适用范围

规则适用于简单元素标签：

- 用户把端点标签输入或快捷替换为元素符号时，例如 `N`、`O`、`Cl`。
- 已经由 engine 生成过的同元素加氢标签，例如 `NH3`、`NH2`、`OH`。
- 标签必须仍然能识别为当前节点元素；任意文本、缩写和基团标签不参与隐式氢刷新。

适用边界：

- `C` 节点按骨架碳处理，`numHydrogens` 保持 `0`。
- `H`、`D` 自身作为终止原子处理。
- `Me`、`Ph` 这类缩写由 functional group 规则处理。
- `CN`、`NO2`、`SO3H` 等基团语义由缩写或价键 parser 处理。
- 自由基语义由电荷/自由基符号规则处理。

## 触发时机

隐式氢刷新发生在 attached node label 几何刷新路径中，主要覆盖：

- 文本框输入端点标签并提交。
- hover 端点后使用键盘快捷替换元素标签。
- 加键、改键导致连接数变化后，相关标签重新排版。

如果修改 Rust engine 中的规则，必须按项目规则重建 Web engine WASM，见 `docs/project-rules.zh-CN.md`。

## 通用计算

对可参与隐式氢的节点：

```text
connection_count = 所有相连键的 bond.order 之和，order 至少按 1 计
abs_charge = abs(node.charge)
numHydrogens = typical_valence - radical_count - connection_count - abs_charge
```

结果会夹在 `0..=9`。

显示文本规则：

- `numHydrogens == 0`：显示元素本身，例如 `N`。
- `numHydrogens == 1`：显示 `元素H`，例如 `OH`。
- `numHydrogens >= 2`：显示 `元素Hn`，例如 `NH2`、`NH3`。

显示方向仍由现有 label 方向规则决定。例如右侧连接时，渲染文本可能显示成 `H2N`，但源文本仍是 `NH2`，重新编辑时按源文本进入编辑器。

## 支持元素

| 元素 | 原子序数 | 典型价态规则 | 示例 |
| --- | ---: | --- | --- |
| `B` | 5 | 默认 3；电荷 `-1` 时 4 | 孤立 `B` -> `BH3`；`B-` 可到 4 价 |
| `N` | 7 | 电荷 `+1` 时 4；电荷 `+2` 或当前连接/电荷合计不超过 3 时 3；否则 5 | 孤立 `N` -> `NH3`；单键 `N` -> `NH2` |
| `P` | 15 | 中性时按 3/5 价阶梯：单键加二氢，双键加一氢，三键不加，四键加一氢，五键不加 | 孤立 `P` -> `PH3`；单键 `P` -> `PH2`；四键 `P` -> `PH` |
| `O` | 8 | 默认 2；正电荷时 3 | 孤立 `O` -> `OH2`；单键 `O` -> `OH` |
| `S` | 16 | 中性时按 2/4/6 价阶梯：单键加一氢，双键不加，三键加一氢，四键不加，五键加一氢，六键不加 | 孤立 `S` -> `SH2`；单键 `S` -> `SH`；五键 `S` -> `SH` |
| `F` | 9 | 1 | 孤立 `F` -> `FH`；单键 `F` -> `F` |
| `Cl` | 17 | 孤立加一氢；单键不加，双键加一氢，三键不加，四键加一氢，五键不加，六键加一氢，七键及以上不加 | 孤立 `Cl` -> `ClH`；单键 `Cl` -> `Cl`；双键 `Cl` -> `ClH` |
| `Br` | 35 | 同 `Cl` | 孤立 `Br` -> `BrH`；双键 `Br` -> `BrH` |
| `I` | 53 | 同 `Cl` | 孤立 `I` -> `IH`；双键 `I` -> `IH` |
| `Si` | 14 | 4 | 孤立 `Si` -> `SiH4` |

## 元素标签识别

用于端点替换的简单元素符号包括：

```text
C, H, N, O, S, P, F, Cl, Br, I, Si, Na, B, D
```

隐式氢只由上表定义。其他可替换元素的行为：

- `C` 被识别为碳节点，按骨架碳显示。
- `H` 和 `D` 被识别为一号原子相关标签，作为终止原子显示。
- `Na` 能作为元素标签替换，但没有隐式氢规则。

## 锚点规则

自动生成的氢字符可以被 hover 高亮，表现上和标签里的普通字符一致；但它不能成为画键锚点。用户从生成的 `H` 字符位置拖出键时，engine 会把实际锚点改回对应重原子，例如 `NH3` 的 `N`。

这个规则只限制画键锚点，不限制文本编辑。生成的氢仍然进入 label source runs，因此重新打开文本框时可以正常编辑。
