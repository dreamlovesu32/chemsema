# ChemSema v1.0.0-beta.2

这个 beta 版本汇总了最近一轮桌面端测试中完成的编辑体验修复和化学语义修复。它比最初的
beta.2 草稿范围更大：括号重复单元、选择状态清理、hover 稳定性、泛基团摘要，以及桌面端/浏览器端交互细节都包含在内。

## 重点

- 括号重复单元计数现在使用明确的 link 模型。选中一个括号和一个文本对象时，可以通过右键菜单或 `Ctrl+L` / `Ctrl+Shift+L` 建立或取消 link；导入 CDXML 时也会根据几何位置和 CDXML 元数据自动配对重复计数标签。
- 已 link 的纯数字括号计数会在重复单元定义明确时参与分子式和分子量计算。取消 link 会解除计数语义，但括号本身仍保留在分子选择模型中。
- 括号绘制生成的文本框按普通编辑对象处理：空标签会在下一次工具动作时消失，非空标签会在切换工具前提交，文本工具可以在创建后继续聚焦和编辑括号标签。括号标签的位置和 7.5 pt 默认字体处理已按 ChemDraw 括号 fixture 调整。
- 选择状态清理更稳定：从 Select 切到其他工具会清除选中状态；双击分子会选中包围它的括号和已 link 的重复计数文本；group 内的普通文本仍可编辑，同时不破坏 group 的整体选择行为。
- 绘制对象、修改弯箭头弧度、以及鼠标在选中对象内部移动后，hover/focus 不再黏附。选中框处于活动状态时，已选中的标签、原子和键会抑制内部 hover 表现。
- 泛基团标签现在不会给出误导性的化学摘要。选中分子中含有 `R`、`R'`、`R''` 或已连接 `Ar` 时，ChemSema 不显示分子式或分子量，因为这类组成在语义上是不确定的。已连接的 `Ar` 在结构标签编辑中按芳基泛基团处理，不再当作氩元素。
- 桌面端和浏览器端的交互细节也做了整理：桌面窗口上沿在弹窗状态下也可以拖动；浏览器编辑区域会拦截原生右键菜单和常见浏览器快捷键；右键菜单的乱码指示符已替换为稳定 ASCII 字符。
- 浏览器 WASM engine、Windows 桌面端可执行文件和安装包已基于同一套修正后的 Rust engine 重建。

## 验证

本 release 使用以下命令验证：

```bash
npm ci
cargo test
npm run verify
```

回归覆盖现在包括括号 CDXML 导入、重复单元 link 计数、group/文本编辑、选中对象 hover 抑制、泛基团化学摘要、完整缩写展开摘要，以及公开的 `kuohao` 括号 fixture。Windows 安装包包含重建后的桌面端可执行文件，以及桌面工作流使用的 Office/OLE 集成服务。
