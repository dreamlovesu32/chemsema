# ChemSema v1.0.0-beta.2

This beta collects the editing-behavior fixes and chemistry-semantics work from
the latest desktop testing round. It is a broader release than the original
beta.2 draft: bracket repeat units, selection cleanup, hover stability, generic
label summaries, and desktop/browser polish are all included.

## Highlights

- Bracket repeat-unit counts now have an explicit link model. A selected bracket
  and text object can be linked or unlinked from the context menu or with
  `Ctrl+L` / `Ctrl+Shift+L`, and imported CDXML repeat-count labels are paired
  automatically when the geometry and CDXML metadata identify the same unit.
- Linked numeric bracket counts participate in formula and molecular-weight
  summaries when the repeating unit is well-defined. Unlinking detaches the
  count semantics while leaving the bracket itself part of the molecule
  selection model.
- Bracket-created text behaves like an editor object: empty labels disappear on
  the next tool action, non-empty labels commit before switching tools, and the
  text tool can focus and edit bracket labels after creation. Default bracket
  label placement and 7.5 pt font handling were tuned against ChemDraw bracket
  fixtures.
- Selection state is cleaned up more aggressively. Switching away from Select
  clears the selection, double-click molecule selection includes enclosing
  brackets and linked repeat counts, and grouped scene text remains editable
  without breaking group behavior.
- Hover/focus state no longer sticks after drawing, curved-arrow curvature
  edits, or pointer movement inside selected objects. Selected labels, atoms,
  and bonds suppress their internal hover affordances while the selection box is
  active.
- Generic labels now keep chemistry summaries honest. If a selected molecule
  contains `R`, `R'`, `R''`, or connected `Ar`, ChemSema does not show formula
  or molecular-weight values because the composition is intentionally
  indeterminate. Connected `Ar` is treated as an aryl generic label rather than
  argon during structure-label editing.
- Desktop and browser interaction details were tightened: the top edge of the
  desktop window can start dragging even while modal prompts are open, the
  browser surface intercepts native context-menu and common browser shortcuts
  during editing, and corrupted right-click menu indicators were replaced with
  stable ASCII glyphs.
- The browser WASM engine, Windows desktop executable, and installer were
  rebuilt from the same corrected Rust engine.

## Validation

The release is validated with:

```bash
npm ci
cargo test
npm run verify
```

Regression coverage now includes bracket CDXML imports, linked repeat-unit
counts, group/text editing, selected-object hover suppression, generic-label
chemistry summaries, complete abbreviation expansion summaries, and the public
`kuohao` bracket fixtures. The Windows installer also includes the rebuilt
desktop executable and the Office/OLE integration service used by the desktop
workflow.
