# ChemSema v1.0.0-beta.7

这个版本把 ChemSema 面向 agent 的 CLI 从一组分散命令推进为对象级操作层。
agent 现在可以定位一个文档对象，检查它的结构和局部视觉上下文，执行受范围约束的
事务式修改，并通过结构化 diff 和 provenance 验证结果。

## 重点

- 新增 `chemsema-cli bundle`，用于生成对象级工作单元。bundle 包含目标 detail、
  context、确定性视觉截图、只包含目标范围的可编辑子文档、identity map、
  provenance，以及明确区分 editable scope 和 visual scope 的 manifest。
- 新增 `chemsema-cli diff`，按 ChemSema id 和字段路径做结构化文档比较，不再只是
  比较 JSON 文本。
- 新增事务式 command-script envelope，支持 preconditions、editable scope、
  dry-run/atomic 执行、创建/删除策略和 postconditions。
- 扩展 JSONL session，支持 bundle 和 transaction 工作流。
- 增强 bundle provenance 和 identity-map artifact，同时避免持久化本地绝对路径。
- 新增真实对象级示例 `examples/agent/07-object-grounded-edit`，基于 `figure1.cdxml`
  对一个 molecule target 完成 bundle、dry-run、编辑、diff、capture 和导出。
- 重构 README 定位，把 ChemSema 更明确地描述为可编辑化学文档平台，并突出同一套
  对象身份如何贯穿可视化编辑、Office 工作流、CLI 检查、受范围约束的 agent 修改、
  验证和可编辑导出。
- 改进真实论文 CDXML 图的渲染保真度，包括键相交时的 margin knockout、基于导入
  `MarginWidth` 的 glyph 级标签退让，以及按 ChemDraw 校准的 attached atom label
  行锚点，覆盖 prime/apostrophe 标签和上下标文本。
- 刷新 README 编辑器界面截图，并基于 `figure1.cdxml` 和 `figure2.cdxml` 重新生成
  ChemDraw/ChemSema 对比素材。

## 验证

本版本使用以下命令验证：

```powershell
npm ci
cargo build -p chemsema-office -p chemsema-cli --release
cargo test
npm run verify
npm run desktop:build
```

同时确认 GitHub Actions CI 在 `main` 和 release tag 上均通过。
