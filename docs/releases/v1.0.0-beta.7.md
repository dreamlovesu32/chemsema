# ChemCore v1.0.0-beta.7

This release turns ChemCore's agent-facing CLI from a collection of useful
commands into an object-grounded operation layer. Agents can now identify one
document object, inspect its structure and local visual context, make scoped
transactional edits, and verify the result with structured diffs and
provenance.

## Highlights

- Added `chemcore-cli bundle` for object-grounded work units. A bundle contains
  target detail, context, a deterministic visual capture, a target-only
  editable subset, an identity map, provenance, and a manifest that separates
  editable scope from visual scope.
- Added `chemcore-cli diff` for structured document comparison by ChemCore ids
  and field paths instead of raw JSON text.
- Added transactional command-script envelopes with preconditions, editable
  scope, dry-run/atomic execution, create/delete policy, and postconditions.
- Extended JSONL sessions with bundle and transaction workflows.
- Added stable provenance and identity-map artifacts for bundle outputs without
  persisting local absolute paths.
- Added a real object-grounded example at
  `examples/agent/07-object-grounded-edit`, using `figure1.cdxml` to bundle,
  dry-run, edit, diff, capture, and export one molecule target.
- Refocused the README around editable chemical documents and shared object
  identity across the visual editor, Office workflows, CLI inspection, scoped
  agent edits, validation, and editable export.
- Improved CDXML rendering fidelity for published reaction figures, including
  bond crossing margin knockouts, glyph-based label retreat from imported
  `MarginWidth`, and ChemDraw-calibrated line anchors for attached atom labels,
  including prime/apostrophe labels and script text.
- Refreshed the README editor screenshot and regenerated the published
  ChemDraw/ChemCore comparison assets from `figure1.cdxml` and `figure2.cdxml`.

## Validation

This release was validated with:

```powershell
npm ci
cargo build -p chemcore-office -p chemcore-cli --release
cargo test
npm run verify
npm run desktop:build
```

GitHub Actions CI was also checked for both `main` and the release tag.
