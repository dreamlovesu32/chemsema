# 化学语义内核

ChemSema 以 Rust 化学 crate 作为分子图意义的唯一权威。查看器、桌面宿主、CLI 和格式
适配器都使用同一份结果，不得各自实现另一套价态、芳香性、氢或规范化规则。

## 清理流程

`chemsema-chemistry::sanitize` 会在 SMILES 改变文档或生成标识符之前依次执行：

1. 检查原子引用、自连接键和重复键；
2. 检查芳香键，并要求每条芳香键都属于环；
3. 为受支持的芳香原子寻找确定性的 Kekulé 匹配；
4. 计算键级价态，拒绝当前模型不支持的超价状态；
5. 结合元素、电荷、芳香状态、显式氢和价态阶梯计算隐式氢；
6. 检查方向键标记，并把完整的烯烃方向组合归一化为 E/Z；
7. 保存 `@`/`@@` 的 OpenSMILES 配体顺序，检查四配位，并在当前四面体 CIP 比较器能够
   区分四个配体时指派 R/S。

配位键不占用普通共价价态。过渡金属氧化态和电子计数属于独立语义层；系统绝不会为了
让下游算法接受而把配位键近似成单键。

当前价态模型覆盖常见有机元素、带电 B/N/O、Si、扩展价态 P/S 和卤素价态阶梯。其他
元素仍可表示，但不会被虚假标记为“已经完成完整价态验证”。

## Canonical SMILES

ChemSema 使用迭代图细化和有上限的对称类个体化计算与输入编号无关的原子排序，再据此
写出组分和环闭合。四面体奇偶性会按照最终写出时的邻体顺序重写（包括环闭合配体），
归一化 E/Z 也会获得确定的斜线方向，因此返回 `canonical: true`。

Canonical SMILES 本来就是工具包相关的。ChemSema 保证的是当前算法和协议版本内稳定，
不保证字符串与 RDKit 或其他工具包逐字节相同。搜索上限可以避免高度对称病理图阻塞编辑器；
如果超过上限，系统会返回异构但非 canonical 的 SMILES，同时给出
`canonical: false` 和 `canonicalReason`，不会冒充规范结果。

## 立体化学边界

双键两侧相邻单键上的方向标记会被作为整体解释。例如 `F/C=C/F` 归一化为 E，
`F/C=C\F` 归一化为 Z；不邻接双键的孤立方向标记会被拒绝。四面体中心保留 SMILES
`@`/`@@` 奇偶性和 OpenSMILES 配体顺序，并要求四个配体。等价遍历会被重写成同一个
canonical isomeric SMILES。对于当前递归 CIP 比较能够区分四个配体的普通四面体中心，
sanitizer 会报告 R/S。这里不冒充完整的 2013 CIP 系统：伪不对称、穿过其他立体单元的
完整序列规则递归、轴/累积双键手性、平面四方、三角双锥和八面体描述符仍不支持。

归一化后的 E/Z 会先投影成确定的相对二维几何，再交给官方 InChI；原生回归测试要求 E
和 Z 生成不同的 InChI `/b` 立体层。四面体中心会投影成一根与构型一致的 V2000 楔键；
原生回归测试以 L/D-丙氨酸核对官方 InChI 的 `/t` 与 `/m` 层。SMILES 导入在 GUI 中画出
同一根楔键；即使没有源 SMILES 奇偶元数据，结构分析也能依据楔键和当前二维几何还原中心。
配位键仍会被拒绝，
直到金属断键和重连层策略明确。

## 分析元数据

`chemistry` 命令继续用 `value` 返回可复制标识符，并按情况附带：

- `canonical`、`canonicalReason` 和 `isomeric`；
- 每个原子的 `implicitHydrogens`；
- 归一化的 `doubleBondStereo` 和验证后的 `tetrahedralCenters`，可指派时含 `cip`；
- `properties.formula`、`formalCharge`、原子数和组分数。

导入语义元数据只是来源记录，不是永久覆盖。用户改变原子、形式电荷或芳香键显示键级后，
分析会退回当前可编辑分子图重新判断。
