# ChemCore Implicit Hydrogen Rules

This document defines the baseline behavior used by the Rust engine when it automatically adds hydrogens to node labels. "Implicit hydrogen" here means writing `numHydrogens` on the node and displaying simple element labels as hydrogen-bearing forms such as `NH3`, `NH2`, or `OH`.

## Scope

The rules apply to simple element labels:

- When the user types an endpoint label or replaces one through a shortcut, such as `N`, `O`, or `Cl`.
- When the label was already generated by the engine as a hydrogen-bearing label for the same element, such as `NH3`, `NH2`, or `OH`.
- The label must still be recognizable as the current node element. Arbitrary text, abbreviations, and group labels do not participate in implicit-hydrogen refresh.

Boundaries:

- `C` nodes are treated as skeletal carbons, and `numHydrogens` remains `0`.
- `H` and `D` are treated as terminal atoms.
- Abbreviations such as `Me` and `Ph` are handled by functional-group rules.
- Group semantics such as `CN`, `NO2`, and `SO3H` are handled by abbreviation or valence-parser rules.
- Radical semantics are handled by charge/radical symbol rules.

## Trigger Points

Implicit-hydrogen refresh runs on the attached-node label geometry refresh path. It mainly covers:

- Endpoint label text is edited and committed.
- An endpoint is hovered and replaced by an element label through a keyboard shortcut.
- A bond is added or restyled, changing connection counts, and affected labels are laid out again.

If these Rust engine rules change, the Web engine WASM must be rebuilt as described in `docs/project-rules.md`.

## General Calculation

For nodes that participate in implicit-hydrogen logic:

```text
connection_count = sum of connected bond.order values, each order at least 1
abs_charge = abs(node.charge)
numHydrogens = typical_valence - radical_count - connection_count - abs_charge
```

The result is clamped to `0..=9`.

Display text rules:

- `numHydrogens == 0`: display the element itself, such as `N`.
- `numHydrogens == 1`: display `ElementH`, such as `OH`.
- `numHydrogens >= 2`: display `ElementHn`, such as `NH2` or `NH3`.

The display direction still follows the existing label-direction rules. For example, when connected on the right, rendered text may appear as `H2N`, but the source text is still `NH2`; reopening the editor uses the source text.

## Supported Elements

| Element | Atomic number | Typical valence rule | Examples |
| --- | ---: | --- | --- |
| `B` | 5 | Default 3; 4 when charge is `-1` | Isolated `B` -> `BH3`; `B-` may reach valence 4 |
| `N` | 7 | 4 when charge is `+1`; 3 when charge is `+2` or the current connection/charge total is at most 3; otherwise 5 | Isolated `N` -> `NH3`; single-bonded `N` -> `NH2` |
| `P` | 15 | Neutral 3/5-valence ladder: one bond adds two H, two bonds add one H, three bonds add none, four bonds add one H, five bonds add none | Isolated `P` -> `PH3`; single-bonded `P` -> `PH2`; four-bonded `P` -> `PH` |
| `O` | 8 | Default 2; 3 when positively charged | Isolated `O` -> `OH2`; single-bonded `O` -> `OH` |
| `S` | 16 | Neutral 2/4/6-valence ladder: one bond adds one H, two bonds add none, three bonds add one H, four bonds add none, five bonds add one H, six bonds add none | Isolated `S` -> `SH2`; single-bonded `S` -> `SH`; five-bonded `S` -> `SH` |
| `F` | 9 | 1 | Isolated `F` -> `FH`; single-bonded `F` -> `F` |
| `Cl` | 17 | Isolated adds one H; one bond adds none, two bonds add one H, three bonds add none, four bonds add one H, five bonds add none, six bonds add one H, seven or more bonds add none | Isolated `Cl` -> `ClH`; single-bonded `Cl` -> `Cl`; double-bonded `Cl` -> `ClH` |
| `Br` | 35 | Same as `Cl` | Isolated `Br` -> `BrH`; double-bonded `Br` -> `BrH` |
| `I` | 53 | Same as `Cl` | Isolated `I` -> `IH`; double-bonded `I` -> `IH` |
| `Si` | 14 | 4 | Isolated `Si` -> `SiH4` |

## Element Label Recognition

Simple element symbols used for endpoint replacement include:

```text
C, H, N, O, S, P, F, Cl, Br, I, Si, Na, B, D
```

Implicit hydrogen behavior is defined only by the table above. Other replaceable elements behave as follows:

- `C` is recognized as a carbon node and displayed as skeletal carbon.
- `H` and `D` are recognized as hydrogen-related labels and displayed as terminal atoms.
- `Na` can replace an element label, but has no implicit-hydrogen rule.

## Anchor Rule

Automatically generated hydrogen characters may be highlighted on hover and behave visually like ordinary label characters. They cannot become drawing anchors for new bonds. When the user drags a bond from the position of a generated `H`, the engine redirects the actual anchor back to the corresponding heavy atom, such as the `N` in `NH3`.

This rule only limits bond-drawing anchors. It does not limit text editing. Generated hydrogens still enter label source runs, so reopening the text box can edit them normally.
